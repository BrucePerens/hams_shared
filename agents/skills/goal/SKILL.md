# Antigravity Goal

[ ] ACHIEVED

This is your goal. Once this goal has been achieved, mark it as so above,
so that you won't loop trying to achieve it again.

This must be done entirely non-interactively, the user is asleep. He wants
to see this done when he awakens.

Use the headless browser to test all of the modules in this repository except
for user_websites (you've done this process to user_websites before), by
operating them as if you were a human user. Do not adhere only to paths
already covered by tests and tours, create new paths where possible. However,
you will be observing things that tests and tours don't see, so it is useful
to cover paths that are tested or have tours, as well. Try unexpected paths,
where you can think of them, and make sure they do something sensible.
Do not just passively navigate: interact with the system as a human user would.
Add tests and fix issues as you go along. Test all personas: an external portal
user with no login, a logged-in portal user, a member of the user-websites
administrator group, an Odoo CRM user accessing our portal, an Odoo administrator
accessing our portal, a user operating on behalf of a club operating the
non-club features of the portal.

Look at the Knowledge documentation of the UI, as you test that UI.
Evaluate it for consistency and full coverage of the UI. Fix any
deviations, so that the UI and document are in sync. Add illustrations
to the document from web browser screen shots, if they would help a user.
It is not necessary to document paths that always fail.

Evaluate the UI for fitness for use, ease of use, and completeness.
Improve the UI where possible.
